# DreamChord 安全说明

DreamChord 是本地优先的创作工具，但仍会处理账号、故事、上传文件和作者配置的模型密钥。本文说明当前安全边界和问题报告方式。

## 支持范围

当前维护版本为 `0.2.x`。仓库尚未承诺长期支持周期；安全修复优先进入最新版本，并在 `CHANGELOG.md` 记录用户需要采取的操作。

## 报告安全问题

涉及认证绕过、越权读取、API Key 泄漏、任意文件访问、恶意上传、服务端请求伪造或备份导入绕过时，请使用 GitHub 仓库的 **Security -> Report a vulnerability** 私密报告入口，不要先创建公开 Issue，也不要上传真实密钥、用户数据库或可识别个人信息。

普通界面错误、功能建议和不含敏感信息的崩溃可以使用公开 Issue。

报告中请包含：

- 受影响版本或提交；
- 最小复现步骤；
- 预期与实际结果；
- 影响范围；
- 已脱敏的日志或请求示例；
- 是否已在公开环境披露。

## 数据保存与网络

- 默认 SQLite 数据库位于 `apps/server/prisma/dev.db`。
- 上传文件保存在本地上传目录。
- 除作者主动配置的模型请求外，故事与素材默认不发送给第三方模型服务。
- 外部模型提供商可能按其条款处理请求内容，作者应避免发送不允许离开本机的素材。
- 项目导出文件包含故事和实际引用的素材，应按创作原稿保管。

## 密钥处理

- `.env` 和 API Key 不得提交到 Git。
- API Key 不应出现在截图、Issue、对话正文、运行日志或项目备份中。
- 怀疑密钥泄漏时应立即在对应提供商撤销并重新生成。
- 演示账号只用于本地演示，不应直接用于公网部署。

## 上传文件边界

服务端使用 Sharp 或文件签名检查真实媒体内容，并限制大小、尺寸和格式。原始上传与衍生候选使用服务端控制的路径。被角色或故事引用的素材不能直接删除。

这些防护降低风险，但不代表 DreamChord 是通用恶意文件分析器。不要向公开实例上传来源不明的文件。

## Agent 安全边界

Agent 只能调用注册的 DreamChord 业务工具，不具有任意 shell、SQL、文件系统或浏览器权限。剧情和素材行动先生成提案，必须经过校验和作者确认。

新增工具时必须同时实现 schema、授权、资源限制、输出脱敏和测试。不要为了提升“自主性”绕过审批、审计或撤销机制。

## 公网部署提示

0.2.0 主要面向本地运行。准备公网部署前至少需要重新评估：

- 强密码、账号注册与速率限制；
- HTTPS、反向代理和安全响应头；
- JWT 密钥轮换与会话失效；
- 上传隔离、恶意文件扫描和对象存储策略；
- 数据库备份、恢复演练和访问审计；
- 外部模型费用、配额和滥用控制；
- CORS、可信代理和服务端请求伪造防护配置。

不要把本地演示配置直接暴露到互联网。
