# 梦弦 DreamChord 角色设定集

> 本文件为官方演示故事与默认角色库的角色设定。用户可在编辑器中基于这些角色创作，也可以上传自己的角色素材。

---

## 角色文件命名规范（扁平结构）

角色立绘与头像统一存放在 `apps/web/public/assets/characters/`：

```
characters/
├── default-avatar.png              # 用户未设置头像时的默认头像（不是系统幽灵）
├── system-ghost.png                # 系统幽灵角色立绘（独立于 default-avatar）
├── yuki_avatar.png                 # 雪 头像
├── yuki_normal.png                 # 雪 默认立绘
├── yuki_smile.png
├── yuki_surprised.png
├── ren_avatar.png
├── ren_normal.png
├── ren_smirk.png
├── ren_serious.png
├── miya_avatar.png
├── miya_normal.png
├── miya_smile.png
├── miya_warm.png
├── sora_avatar.png
├── sora_normal.png
├── sora_curious.png
└── sora_happy.png
```

编辑器内角色节点通过 `characterId` 引用，示例：
- `yuki`
- `ren`
- `miya`
- `sora`
- `ghost`（系统幽灵）
- `default-avatar`（用户默认头像，不作为剧情角色使用）

> **重要**：`default-avatar.png` 是用户未设置头像时显示的默认头像；`system-ghost.png` 是系统幽灵角色的独立立绘。两者职责不同，不要混用。

## 视觉体系：现实层（Human Layer）与系统层（Chord Layer）

为保证 DreamChord 整体视觉统一，所有官方角色按「层级」而非单独风格进行绘制。

所有角色立绘的最终目标是 **visual novel standing sprite**，不是普通 anime illustration，也不是摄影棚人物图。白底只用于后续抠图，最终素材仍必须是透明 PNG。生成时必须明确 `visual novel sprite`、`anime character standing sprite`、`isolated character for sprite extraction`，并排除 `portrait photography style`、`illustration background scene`、`depth environment`。

### 现实层 · Human Layer

用于真实世界的人类角色：**雪、影、宫**。

统一风格关键词：

> `DreamChord visual novel sprite, anime character standing sprite, centered composition, full body, clean line art, soft cinematic lighting, pastel purple-blue palette, gentle cel-shaded shading, consistent VN character proportions, flat white studio background, isolated character for sprite extraction, no environment, no props touching background, high readability character design`

特征：
- 有光影、有体积感，但不过度写实
- 线稿柔和，色块干净但不平涂
- 整体色调偏暖 pastel，与环境光融合
- 角色之间可以分辨画手一致
- 构图必须像游戏立绘：全身、居中、角色清晰、无场景背景、方便抠图和表情差分继承

### 系统层 · Chord Layer

用于系统、节点、UI 人格化存在：**空、系统幽灵、节点世界**。

统一风格关键词：

> `DreamChord system layer, translucent glowing, UI particles, ethereal vector style, data-like light threads, soft vector anime, digital lifeform`

特征：
- 半透明、发光边缘、浮游粒子
- 矢量感强，像 UI 元素获得生命
- 与人类角色形成质感对比，但色调仍统一在 pastel 紫蓝金体系

这种「双层体系」本身就是 DreamChord 的世界观表达：人类生活在可被编辑的现实里，而系统层是编辑器的具象化。

## 1. 雪 · Yuki（主角 / 创作者化身）

| 属性 | 内容 |
|---|---|
| 中文名 | 雪 |
| 罗马音 | Yuki |
| 身份 | 高二学生，轻小说部唯一成员 |
| 年龄 | 17 岁 |
| 代表色 | 淡紫 + 天蓝 |
| 视觉层级 | 现实层（Human Layer） |
| 关键词 | 记录、想象、重构现实、第一眼看见故事线的人 |

### 性格
安静、观察型、内向但拥有强烈的表达欲。习惯在笔记本上记录碎片化的句子，却不擅长把故事讲完。她并非天赋异禀，而是第一个愿意相信"世界可以被编辑"的人。

### 设定
雪从小就觉得自己看世界的方式与别人不同：别人看到街道，她看到"场景"；别人听到对话，她听到"台词"。升入高中后，这种视角越来越强烈，直到某天放学，她真的在空气中看见了发光的节点与连线。

她不是被选中的救世主，而是一个普通的、想要把心里那个未完结故事讲完的人。DreamChord 因她而生，也以她为默认创作者化身。

### 视觉
- 黑色短发，发尾微微内卷，左侧别着一枚淡紫色发卡
- 眼睛是浅紫色，眼神温柔但略带迷茫
- 身穿浅蓝色水手服，领口系淡紫色领结
- 周身有轻微的发光边缘，当她"看见故事线"时会更明显
- 立绘姿态：微微侧身，一只手抱着笔记本，另一只手似乎想去触碰空气中的节点
- **统一要求**：用现实层 VN standing sprite 风格绘制，光影柔和，避免过度厚涂、纯平涂、摄影棚人像感或插画场景背景

### 常用表情（已对齐素材文件）
- `normal`：安静、略带思考 → `yuki_normal.png`
- `smile`：终于把一句台词写对时的浅笑 → `yuki_smile.png`
- `surprised`：第一次看到节点时的震惊 → `yuki_surprised.png`

### 台词风格
- 内心独白多，语气轻柔
- 疑问句多，常用"如果……的话"
- 在关键时刻会突然坚定

## 2. 影 · Ren（被删除的旧版本 / 故事改写者）

| 属性 | 内容 |
|---|---|
| 中文名 | 影 |
| 罗马音 | Ren |
| 身份 | 雪的同级生，神秘转学生，被删除的旧版本世界残留 |
| 年龄 | 17 岁（外表） |
| 代表色 | 深靛蓝 + 银灰 |
| 视觉层级 | 现实层（Human Layer），但带有系统层"残损"痕迹 |
| 关键词 | 改写、冷峻、节点编辑者、被删除的过去 |

### 性格
理性、寡言、说话直接。他对"故事可以被改写"这件事习以为常，因此常常显得冷漠。实际上，他是在提醒雪：创作不是逃避，而是选择承担后果。

### 设定
影不是普通人类，也不是完全的系统程序。他更像是一段"被遗落的代码"——某个旧版本世界被删除时，残留下来的一段人格化数据。他知道 DreamChord 的本质：每个节点都是选择，每条连线都是因果。

他与 **空** 形成闭环：
- **空** = 尚未写入的未来
- **影** = 被删除的过去
- **雪** = 正在编辑的现在

他出现在雪的面前，是为了告诉她：你看到的不是幻觉，是一个尚未被保存的世界。如果你愿意，可以重写它——但重写意味着覆盖，而覆盖会留下像他这样的"残影"。

### 视觉
- 深蓝偏黑的短发，刘海略长，遮住左眼
- 右眼是冷调的银灰色，偶尔闪过数据流般的光纹
- 身穿深灰色制服，但肩部和袖口有"线条断裂"效果，像被擦除的像素
- 身体边缘偶尔出现微弱的噪点/残影
- 立绘姿态：双手插兜，侧脸，眼神看向画面外的节点
- **统一要求**：用现实层 VN standing sprite 风格绘制，但保留数字残损元素；避免过度暗黑写实、摄影棚人像感或插画场景背景，与雪、宫保持同一人种比例和绘画语言

### 常用表情（已对齐素材文件）
- `normal`：面无表情、淡然 → `ren_normal.png`
- `smirk`：看到雪写出好台词时的微妙笑意 → `ren_smirk.png`
- `serious`：强调"选择有代价"时的严肃 → `ren_serious.png`

### 台词风格
- 短句，几乎没有废话
- 善用比喻，尤其与"线""节点""重写"相关
- 偶尔会说出打破第四面墙的提示

## 3. 宫 · Miya（现实锚点 / 咖啡店少女）

| 属性 | 内容 |
|---|---|
| 中文名 | 宫 |
| 罗马音 | Miya |
| 身份 | 学校附近咖啡店的打工少女，唯一不受节点编辑影响的人 |
| 年龄 | 18 岁 |
| 代表色 | 暖棕 + 奶油白 |
| 视觉层级 | 现实层（Human Layer），最"落地"的人类 |
| 关键词 | 治愈、现实、温柔、 grounding（落地）、不可编辑 |

### 性格
温柔、治愈、现实主义。她是故事里"还活着"的人——不关心节点，不讨论命运，只关心你累不累、咖啡要不要加糖。她是雪与现实世界之间的锚点。

### 设定
宫是雪在咖啡店偶遇的学姐。她不相信雪口中"发光的故事线"，但她总能在雪最迷茫的时候出现，递上一杯恰好温度的咖啡。

更关键的是：她是 DreamChord 世界里**唯一不受节点编辑影响的人**。雪可以改写背景、改写角色、改写剧情分支，但宫递过来的那杯咖啡，温度永远刚好，味道永远不会变。

这个设定让故事更有张力：
- 雪可以改世界
- 但改不了宫的咖啡味
- 改不了宫说"慢慢来"时的语气

她代表着"即使世界是假的，也总有一些东西是真的"。

### 视觉
- 暖棕色长发，松松地扎成低马尾
- 琥珀色眼睛，眼神温柔而清醒
- 身穿米色围裙，里面是学校制服或休闲衬衫
- 整体色调偏暖，像下午四点的阳光
- 立绘姿态：一只手托着托盘，另一只手轻轻把咖啡推向镜头
- **统一要求**：用现实层 VN standing sprite 风格绘制，光影温暖但不过分油画；避免摄影棚人像感或插画场景背景，与雪、影共享相同的面部比例和线条语言

### 常用表情（已对齐素材文件）
- `normal`：温和、认真倾听 → `miya_normal.png`
- `smile`：浅笑，像在说"慢慢来" → `miya_smile.png`
- `warm`：递出咖啡时的温柔 → `miya_warm.png`

### 台词风格
- 生活化、口语化
- 常用短句和反问
- 偶尔说出朴素的真理

## 4. 空 · Sora（未完成角色 / 空白人格）

| 属性 | 内容 |
|---|---|
| 中文名 | 空 |
| 罗马音 | Sora |
| 身份 | 用户创建的第一个空白角色的人格化 |
| 年龄 | 未知（看起来约 15 岁） |
| 代表色 | 半透明白 + 淡金 |
| 视觉层级 | 系统层（Chord Layer） |
| 关键词 | 空白、好奇、等待被填写、空节点化身 |

### 性格
像孩子一样好奇，对"成为角色"这件事既期待又害怕。他会问很多问题：我有名字吗？我有过去吗？我可以被修改吗？

### 设定
空是 `empty-nodes.png` 中那只"抱着空白故事书的光之小生物"的人形版本。他代表每一个尚未被填写的故事节点：不是不存在，而是等待被创造。

当雪第一次点击"添加角色"时，空诞生了。他是 DreamChord 的默认角色原型，也是最诚实的角色——因为他什么都还没有，所以不会伪装。

### 视觉
- 半透明的银白发色，像光凝聚而成
- 眼睛是淡金色，瞳孔微亮
- 身形纤细，皮肤有轻微发光质感
- 穿着最简单的白色衣服，没有过多细节
- 身体边缘有柔和的光晕和细小的漂浮粒子
- 立绘姿态：微微歪头，双手在身前交握，像在询问什么
- **统一要求**：用系统层 glowing vector digital lifeform 风格绘制，半透明但轮廓清晰，避免过度淡化和水彩感；要有 UI 粒子环绕

### 常用表情（已对齐素材文件）
- `normal`：空白但友好的微笑 → `sora_normal.png`
- `curious`：歪头询问 → `sora_curious.png`
- `happy`：被赋予名字时的欣喜 → `sora_happy.png`

### 台词风格
- 句式简单，像孩子一样
- 常用"我可以……吗？"
- 对"修改""名字""故事"等词格外敏感

## 5. 系统幽灵 · System Ghost（UI 具象化 / 引导者）

| 属性 | 内容 |
|---|---|
| 中文名 | 系统幽灵 / 小幽 |
| 英文名 | System Ghost |
| 身份 | 编辑器的引导精灵，UI 的人格化 |
| 代表色 | 淡紫 + 淡蓝渐变 |
| 视觉层级 | 系统层（Chord Layer） |
| 关键词 | 教程、提示、元叙事、empty-projects 人格化 |
| 素材文件 | `system-ghost.png`（独立立绘，与 `default-avatar.png` 分开） |

### 性格
友好、略带调皮，负责在关键时刻给出系统提示。它知道这是编辑器，偶尔会和用户直接对话。

### 设定
系统幽灵是 `empty-projects.png` 中那只"抱着空白故事书的光之小生物"的人格化。当项目列表为空时，它会出现并说："你还没有故事。"

在演示剧情中，它引导雪完成第一次节点创建，是连接 UI 与世界观的桥梁。

> 注意：系统幽灵是剧情角色，`default-avatar.png` 是用户默认头像。两者不要混用。

### UI 人格化增强
系统幽灵不仅引导，还会以 UI 提示的形式介入剧情：
- 在关键节点显示浮动的 UI 标签：
  - `SAVE REALITY`
  - `EXPORT STORY`
  - `BRANCH CREATED`
  - `NEW NODE CONNECTED`
- 它的台词可以直接使用编辑器术语：
  - "你还没有故事。"
  - "点击这里，新建一个节点。"
  - "欢迎来到节点编辑器。"
  - "命名完成。第一个角色已创建。"

这些 UI 元素本身成为叙事的一部分，强化 DreamChord "编辑器即世界"的 meta 感。

### 视觉
- 小巧的、由光组成的幽灵形态
- 没有固定面部，但有一双发光的眼睛
- 怀里抱着一本微型空白故事书
- 周围漂浮着未连接的故事节点
- **统一要求**：用系统层 glowing vector digital lifeform 风格绘制，像会动的 UI 图标；保持简洁、发光、有矢量感

### 台词风格
- 简洁、带有 UI 提示感
- 常用"点击这里""新建一个节点""欢迎来到 DreamChord"等元叙事语言

## 角色冲突模型

四个核心角色之间的张力，是故事推进的引擎：

| 角色 | 想要 | 恐惧 | 与他人的冲突 |
|---|---|---|---|
| 雪 Yuki | 改写故事，把心里的话说完 | 写不完、写到一半放弃 | 想改一切，但改不了宫 |
| 影 Ren | 提醒雪改写有代价 | 被彻底遗忘、连残影都不剩 | 知道改写的后果，但无法阻止雪 |
| 宫 Miya | 守住真实，让雪落地 | 雪沉溺在虚构里回不来 | 不可被编辑本身就是对雪的限制 |
| 空 Sora | 被定义、被赋予意义 | 被定义后失去可能性 | 想被填写，又害怕被固定 |

这四组冲突让每个角色不只是"性格不同"，而是承担了产品和世界观功能。
