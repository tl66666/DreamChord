About

把节点图的复杂性隐藏在镜头卡背后

DreamChord 是一个面向非技术用户的视觉小说创作平台。传统视觉小说引擎（如 Ren'Py）要求创作者学习脚本语法、手动管理跳转标签、理解节点连线逻辑，门槛极高。DreamChord 的核心理念是：用户只需像写小说一样填写背景、选择角色、编写台词，系统自动将其转换为可播放的节点图和运行时场景。通过"章节 → 场景 → 镜头卡"三级结构组织剧情，一键发布即可在浏览器中直接播放——无需安装任何运行时，无需部署服务器，分享链接即可阅读。

节点图太难
传统引擎痛点：Ren'Py 需要手写 label 和 jump 语句，TyranoBuilder 的节点连线复杂且容易出错，新手往往在"如何连接两个场景"上卡住数小时。DreamChord 方案：镜头卡写作界面，用户只需填写"谁说了什么"，系统自动创建 background → character → dialogue 节点链并生成连线，用户完全不需要看到节点图。
没有可视化预览
传统引擎痛点：修改一句台词需要重新编译运行整个项目，无法实时看到立绘位置、表情变化和文本框效果。DreamChord 方案：编辑器右栏 MiniPreview 实时渲染，每改一个字、换一个背景、调整一个站位，立刻在右侧看到与最终播放器完全一致的效果。支持"完整预览"全屏沉浸式查看。
分支管理混乱
传统引擎痛点：多分支剧情容易产生"死路"（选择了某选项但忘记写后续）、"断线"（jump 目标不存在）和"无限循环"，调试极其困难。DreamChord 方案：选项无去向时自动显示琥珀色警告徽章和汇总提示条；剧情流程图以 galgame 风格可视化所有场景和分支连线，汇合场景紫色高亮；项目体检面板自动检测 12 项健康指标。
AI 辅助缺失
传统引擎痛点：没有内置 AI 辅助，创作者需要自己在 ChatGPT 和编辑器之间反复切换复制粘贴，上下文容易丢失。DreamChord 方案：编辑器内置 AI 助手面板，五种模式——润色当前台词、续写后续剧情、生成选项分支、批量生成角色回应、AI 直接生成完整节点图。支持 GLM / DeepSeek / Kimi / OpenAI 四家供应商，通过 BaseOpenAICompatibleProvider 统一接口。
无法即时分享
传统引擎痛点：Ren'Py 需要打包为 exe/apk 分发给读者，每次修改都要重新打包；Web 引擎需要自己部署服务器。DreamChord 方案：一键发布到作品广场，生成分享链接，读者打开浏览器即可游玩。支持移动端适配，无需安装任何客户端。
长篇组织困难
传统引擎痛点：超过 50 个场景后，线性列表无法体现剧情结构，找不到"第三章的分支B在哪里"。DreamChord 方案：章节 → 场景 → 镜头卡三级结构，每章独立画布，顶部中文数字标签切换（第一章、第二章…）。章节可折叠，场景树支持搜索。长文导入功能可自动将粘贴的小说正文按"角色：台词"格式拆分为镜头卡。
素材管理割裂
传统引擎痛点：角色立绘、背景图、音乐文件散落在各处，需要手动写路径引用，更换素材需修改代码。DreamChord 方案：内置素材库面板，角色卡片包含立绘、表情差分（normal/smile/serious）、人物小传和戏剧冲突。编辑器中直接选择素材，无需手写路径。支持自定义上传。

Interface

以下均为项目实际运行截图，非设计稿。从编辑器到流程图，从 AI 写作到素材管理，每一处界面都围绕"让创作更简单"设计——隐藏节点图复杂性，暴露小说式写作体验。

localhost:5173/editor
01 / Editor

镜头卡编辑器

三栏布局是整个产品的核心。左侧 SceneTree 管理章节和场景的树形结构——新建章节、新建场景、重命名、删除，全部在树中完成。中间 ShotCardEditor 是用户真正写作的区域，每张镜头卡像一张便签卡片，包含背景选择、角色登场、台词编辑、选项设置。右侧默认显示 MiniPreview 实时预览，也可切换为素材库或 AI 助手面板。所有编辑操作通过 Zustand store 统一管理，自动防抖保存。

  • 章节 → 场景 → 镜头卡三级结构，顶部中文数字标签切换
  • 6 种镜头卡类型：对话、旁白、心理、回忆、系统、选项
  • 右栏实时预览，所见即所得——每改一个字立刻更新
  • 长文导入：粘贴小说正文，自动按"角色：台词"格式拆分
  • 选项无去向时显示琥珀色警告徽章 + 汇总提示条
  • 支持紧凑视图，适合长篇剧情的快速浏览

02 / Flowchart

剧情流程图

当剧情复杂到需要全局视角时，切换到剧情流程图视图。这是 galgame 风格的全局可视化——所有场景以卡片形式排列，按章节分行。普通场景间用实线箭头连接，选项分支用虚线箭头并标注选项文本。汇合场景（接收多条不同场景入边）自动检测并以紫色高亮 + GitMerge 图标标识。支持滚轮缩放、拖拽平移、拖拽场景卡片调整位置、拖拽创建场景间连接。

localhost:5173/ai-writing
03 / AI Writing

AI 写作辅助台

独立的 AI 写作工作台，五种模式覆盖从构思到落笔的全流程。选择任务类型（整理大纲 / 场景续写 / 角色台词 / 选项分支 / 节点草稿），输入当前剧情上下文和辅助需求，AI 生成结果可直接复制或保存到草稿箱。草稿箱持久化保存，随时可复制到编辑器节点。未配置 AI 时使用本地规则引擎生成基础内容，配置后通过 BaseOpenAICompatibleProvider 统一接口调用 GLM / DeepSeek / Kimi / OpenAI。

  • 整理大纲：从零生成故事框架和章节规划
  • 场景续写：根据上下文延续当前剧情走向
  • 角色台词：批量生成特定角色的对话风格
  • 选项分支：自动产出多选项及各自后续方向
  • 节点草稿：AI 直接生成完整 nodes + edges JSON
  • 本地辅助模式 + 云端 AI 模式双轨运行

04 / Assets

素材库

角色立绘、背景图、表情差分统一管理。每个角色卡片采用漏斗式信息层级：立绘全身像 → 一句话定位 → 人物小传 → 故事大纲 → 戏剧冲突（粉色高亮）→ 底部表情差分选择器。戏剧冲突字段用色彩背景突出，降低创作者在大量角色间扫描核心矛盾的成本。支持恢复默认素材和自定义上传，上传的素材通过 customUrl 字段处理非标准路径。

  • 角色立绘 + 表情差分（normal / smile / serious）
  • 背景图库：樱花坡道、教室、咖啡馆、星空
  • 戏剧冲突粉色高亮，快速定位角色核心矛盾
  • 立绘差分底部选择器，一键切换调用
  • 支持自定义上传 PNG/JPG/GIF/WebP 素材

Quick Start

三步启动项目

Node.js 20+ 环境，pnpm 或 npm 均可。默认测试账号 demo / demo123，开箱即用。

01

安装依赖

克隆仓库后安装前后端依赖（pnpm workspace 会自动处理共享依赖）

git clone https://github.com/tl66666/DreamChord.git cd DreamChord && pnpm install
02

初始化数据库

配置 .env 环境变量，运行 Prisma 迁移创建表结构，执行种子数据初始化

cd apps/server cp .env.example .env npx prisma migrate dev && npx prisma db seed
03

启动开发服务器

双击 start-dreamchord.bat 一键启动，或手动分别启动前后端开发服务器

cd apps/web && npm run dev cd apps/server && npm run dev
FAQ

常见问题

关于使用方式、技术选型、素材规范和后续迭代的常见疑问。点击问题展开解答。

DreamChord 和 Ren'Py 有什么区别？我该选哪个？+
Ren'Py 是成熟的视觉小说引擎，适合愿意学习脚本语法的创作者，生态完善、社区活跃、可打包为桌面/移动应用。DreamChord 面向不想写代码的创作者——如果你有故事想法但不擅长编程，想快速在浏览器里搭一个可交互的视觉小说，选 DreamChord。两者不是替代关系，而是面向不同人群的工具。
项目需要安装数据库吗？+
不需要额外安装。后端使用 SQLite 文件数据库（dev.db），Prisma ORM 自动管理。运行 npx prisma migrate dev 会自动创建表结构，npx prisma db seed 会插入演示账号和官方剧情数据。整个数据库就是一个文件，不需要 MySQL / PostgreSQL 等独立数据库服务。
AI 功能是必须的吗？不配置 AI 能用吗？+
AI 功能完全可选。不配置 API Key 时，AI 助手使用本地规则引擎生成基础内容（基于模板和关键词匹配）。配置后才会调用 GLM / DeepSeek / Kimi / OpenAI 的在线接口。基础编辑、预览、发布功能不依赖任何 AI 服务。
角色立绘为什么有白底？怎么处理？+
AI 生成角色立绘时使用白底（flat white studio background），白底只为方便后续抠图。项目使用阶段必须放入透明 PNG。白底母版存放在 source-assets/white-bg-originals/，运行用透明立绘存放在 apps/web/public/assets/characters/。抠图可使用 remove.bg、Photoshop 或 rembg 等工具。灰白格子不是背景，只是图片软件表示"透明"的预览方式。
新建场景后无法编辑背景、角色、台词怎么办？+
这通常是场景起始节点定位错误。系统通过"没有来自同场景入边的节点"来定位起始节点（findSceneStartNode 函数），而非按数组顺序查找。如果遇到此问题，检查场景中是否有多个无入边节点，或是否有跨场景的边误连。FlowEditor 和 ShotCardEditor 中的相关调用均已使用 findSceneStartNode() 替代旧的顺序查找逻辑。
选项分支不出现或跳转不正确？+
检查选项卡是否有未设置去向的琥珀色警告标识。每个选项需要通过"写这条分支"（自动创建新场景）或"跳转到已有场景"设置目标场景。另外，addCard 和 quickAppendDialogue 在场景尾节点已有出边时采用插入模式（断开旧出边 → 插入新节点 → 重新连接原目标），确保新节点不会孤立。
支持移动端播放吗？+
播放器支持移动端自适应。文本框铺满屏幕宽度，选项按钮底部弹出，触摸点击即可推进剧情和选择分支。编辑器由于三栏布局较为复杂，建议在桌面端使用。播放器支持快捷键（Enter/Space 推进、1/2/3 选择选项、H 查看历史、S 跳过、A 自动播放），在移动端通过触摸操作替代。
后续迭代计划是什么？+
已规划的方向包括：BGM 音乐管理系统（支持循环、淡入淡出、场景切换过渡）、语音合成集成（TTS 自动朗读台词）、多语言支持（i18n 国际化）、更丰富的动画效果（角色全身入场动画、屏幕震动、转场特效）、以及协作编辑功能。详细迭代建议参见 docs/AI_HANDOFF.md。
