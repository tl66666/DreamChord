# DreamChord 术语表

这份表统一界面、文档和代码讨论中的词义。

| 术语 | 含义 |
|---|---|
| 项目 | 一部独立作品，包含章节、角色、故事圣经和相关素材引用 |
| 章节 | 作品中的主要内容单元，拥有独立剧情图和版本号 |
| 场景 | 同一时间、地点或叙事目的下的一组连续镜头卡 |
| 镜头卡 | 作者主要编辑单元，表达文字、画面变化、人物动作、选择或跳转 |
| 剧情图 | 由节点和边组成的运行结构，用于表达顺序、分支、跳转和汇合 |
| 舞台状态 | 某条剧情路径上当前生效的背景、CG 和在场人物状态 |
| 登场 / 更新 | 让人物进入舞台，或更新其立绘、表情和站位 |
| 保持 | 沿用上一镜头的人物状态，不额外生成显示事件 |
| 退场 | 明确让人物从舞台隐藏 |
| 故事圣经 | 作者维护的稳定世界观、主题、风格、时间线和禁止事项 |
| 创作 Agent | 能读取 DreamChord 项目上下文并通过受限工具提出可审核操作的助手 |
| 对话 | Agent 工作区里的独立讨论线程，可绑定项目和可选章节 |
| 运行 | Agent 处理一次用户请求的生命周期记录 |
| 工具 | Agent 可调用的白名单业务能力，不等同于任意代码执行 |
| 补丁 | 对剧情图的结构化修改提案，应用前需要校验和作者确认 |
| 记忆 | 带作用域、类型、状态和来源的长期上下文记录 |
| 建议记忆 | Agent 或系统提出、尚未被作者确认为正式设定的记忆 |
| 启用记忆 | 可参与后续 Agent 上下文装配的已确认记忆 |
| 遗忘记忆 | 为审计而保留，但不再提供给 Agent 的记忆 |
| 全局素材库 | 当前账号可跨项目复用的图片和音频集合 |
| 原始素材 | 用户上传且始终保留的源文件 |
| 候选图 | 由处理流程生成、尚未接受为正式素材的衍生文件 |
| 立绘 | 带透明背景、绑定角色和表情、可在舞台上定位的人物图片 |
| CG | 用于重要剧情画面的全屏或主要画面插图 |
| 背景 | 表示持续场景空间的画面资源 |
| 自动保存 | 编辑器将最新章节修订按顺序提交到服务端的机制 |
| 基础版本 | 保存或应用补丁时所依据的章节版本，用于防止旧数据覆盖新数据 |
| 剧情健康 | 对不可达节点、断开选项、错误跳转和其他结构问题的检查结果 |
| 项目备份 | 含项目清单、剧情数据和实际素材字节的可移植导出文件 |
