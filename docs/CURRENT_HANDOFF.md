## 本轮完成内容

### 1. Agent 无 Key 与有 Key 的双路径

没有 API Key 时，系统不再把 Agent 退化成“请先配置模型”的提示。服务端使用本地意图路由和项目快照完成确定性任务：项目/章节/角色/素材盘点、剧情体检、创作知识问答、素材提示词、确定性续写草稿，以及用户正文或已选 Agent 草稿的结构化转换。

配置 GLM、DeepSeek、Kimi、OpenAI 或兼容模型后，模型负责更自然的长文续写、对白润色、开放式改写和复杂剧情规划。两种模式都不能直接写入故事，统一经过提案、校验、预览、作者确认和版本安全撤销。

### 2. 正文到工作台的结构化导入

已选草稿会先过滤“续写草稿”“镜头卡说明”“节点连线”等非故事文字，再按下列标记解析：

- `镜头`：拆分场景组；
- `地点`：设置场景标题并尝试匹配项目背景；
- `旁白`、`画面`、`动作`、`音效`：生成旁白卡；
- `角色`：生成角色对话卡，项目中存在同名角色时补充登场节点；
- `选项`：生成选项节点。

匹配采用保守策略。项目内没有精确匹配的背景或角色时，保留文字与结构，不自动套用默认素材。多镜头导入的回归测试还会把补丁应用到内存图，确认节点和连线可通过领域校验。

主要文件：

- `apps/server/src/agent/localAssistant.ts`
- `apps/server/src/agent/localAssistant.test.ts`
- `apps/server/src/agent/runService.ts`
- `packages/story-domain/src/patch.ts`

### 3. Agent 工作流与模型配置体验

- Agent 可新建、搜索、重命名、置顶、删除独立对话；对话可以不绑定章节，也可以绑定后进入受控创作。
- 模型设置页保留来源页面，配置后可返回原 Agent 对话。
- 设置页区分“未启用”“已启用但缺 Key”“Key 已保存在本机”。这里的“已保存”只表示本地配置存在，不代表模型连通性已经验证。
- 右侧信息区使用“创作参考、故事资料、长期设定”等创作语言，不直接暴露工具策略和原始产物 ID。

### 4. 启动、文档和展示材料

- Windows 启动器不再结束任何已有监听进程；端口被占用时会选择候选端口继续启动。
- 新增真实桌面验收截图 `docs/screenshots/agent-structured-import-1440.png`，展示“选中草稿 -> 清洗/解析 -> 补丁预览 -> 应用或放弃”。
- README、项目概览、Agent 指南、展示页和个人网站 DreamChord 项目卡都已同步无 Key/有 Key 边界和结构化导入能力。

## Agent 的工具与外部访问边界

### 内部 Tool Calling：有，且仅限业务白名单

当前注册 15 个内部工具，运行在 DreamChord 服务端，不是外部插件：

| 分类 | 工具 |
|---|---|
| 项目与剧情读取 | `read_project_brief`、`read_chapter_outline`、`read_scene`、`search_story`、`analyze_story_graph` |
| 对话与记忆 | `read_conversation_context`、`search_memories` |
| 角色与素材读取 | `list_project_assets`、`inspect_asset`、`read_character_profile` |
| 剧情提案校验 | `create_story_patch`、`validate_story_patch` |
| 素材候选处理 | `prepare_character_asset`、`prepare_cg_asset`、`prepare_background_asset` |

每个输入经过 Zod Schema 校验；剧情补丁进一步经 `@dreamchord/story-domain` 校验。工具只能读取授权项目资源或生成候选/提案，不能绕过作者确认写入数据库。

### 外部网络访问：只有两种明确场景

| 类型 | 是否需要 API Key | 实际访问 | 发送内容 | 限制 |
|---|---|---|---|---|
| 模型推理 | 需要 | 用户配置的 OpenAI 兼容 `/chat/completions` 接口 | 当前任务和必要项目上下文 | Provider URL 会做 DNS、私网地址和重定向检查；Key 不写入消息、运行记录或数据库 |
| 公共知识查询 | 不需要 | `https://zh.wikipedia.org/api/rest_v1/page/summary/...` | 清洗后的问题关键词 | 只允许中文 Wikipedia、只读 GET、3.5 秒超时、响应长度限制；不发送故事、记忆或密钥 |

本地图片检查、白底去除、裁边和衍生图由 Sharp 在本机完成，不调用外部图像服务。

### 明确没有提供的能力

Agent 没有 MCP Server 集成，也没有浏览器自动化、Shell、任意文件系统、SQL、任意 HTTP URL 或任意代码执行工具。这样做是为了避免创作 Agent 以“帮用户写剧情”为名越过本地数据与网络边界。

## 关键运行链路

```text
用户消息
  -> 识别任务与绑定范围
  -> 装配项目、章节、场景、角色、素材、对话摘要和记忆
  -> 本地确定性助手 或 OpenAI 兼容模型
  -> 白名单工具调用 / 本地结构化解析
  -> StoryPatch + 图校验
  -> 预览
  -> 作者应用、放弃或撤销
```

普通聊天不会生成剧情补丁；只有绑定章节且存在明确修改意图时，才允许进入受控行动。已选草稿导入为了避免模型重写原文，会强制走确定性本地解析器。

## 已知边界

- 本地续写是项目上下文驱动的确定性草稿，不等同于外部大模型的开放式文风创作。
- 正文解析是结构化标记和规则解析，不是通用语义剧本理解；复杂动作、镜头语言和分支后续仍需作者检查。
- 只有提供了后续场景的选项才能自动连接完整分支；仅导入选项文本时，编辑器会保留选项卡并由作者补充分支去向。
- 素材自动绑定坚持精确匹配，宁可留空也不使用不相关立绘或背景。
- 当前产品定位为本地单用户创作工作台，不包含多人实时协作、通用语义抠图、配音生产线或 BGM 时间线。

## 后续迭代建议

按优先级建议继续做以下工作：

1. 扩展剧本中间表示：支持角色退场、表情、站位、转场、音效、镜头动作和选项后的分支正文，仍保持“不能可靠识别就不自动写入”的策略。
2. 为 Agent 建立评测集：覆盖无 Key 回答、模型工具调用、草稿导入、素材匹配、补丁合法性、失败降级和撤销，避免功能越多越难验证。
3. 增加模型连通性测试入口：保存配置后可由用户主动测试 Provider 可用性、模型名和错误原因，但不要把 API Key 持久化到服务端数据库。
4. 提供素材候选匹配面板：当背景或角色只有近似匹配时，向作者展示候选和理由，由作者选择，不把模糊匹配直接写入故事。
5. 在保留本地单用户边界的前提下，逐步评估 PostgreSQL、对象存储和协作编辑的部署模型；这应作为独立架构改造，不和日常故事编辑功能混在一起。

## 接手检查清单

1. 阅读本文件、`docs/AI_HANDOFF.md` 与 `CLAUDE.md`。
2. 运行 `start-dreamchord.bat`，使用 `demo / demo123` 登录。
3. 在 Agent 中不绑定章节询问项目概况，再绑定章节生成或粘贴一份草稿。
4. 选择“生成工作台场景”，检查补丁预览、素材匹配和放弃/应用入口。
5. 打开编辑器确认导入场景的旁白、台词、选项和流程图结构。
6. 提交前运行完整质量检查，并确认 `.env`、数据库、上传文件和日志没有进入 Git。
