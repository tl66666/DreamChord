# DreamChord 项目路线图

更新日期：2026-07-12。

路线图描述可能的后续方向，不代表承诺时间，也不应作为当前版本能力宣传。已完成内容只记录在 [变更记录](../CHANGELOG.md)。

## 近期：提高创作可靠性

- Agent 评测样例：角色口吻、一致性、分支多样性和工具选择回归集。
- 记忆冲突处理：并排查看互相矛盾或被取代的设定，选择保留、合并或遗忘。
- 更完整的章节版本历史：从单次 Agent 撤销扩展为可查看的修订记录。
- 素材引用视图：从素材反查使用它的项目、角色、表情和镜头。
- 新手引导与空状态：让第一次打开项目的用户更快完成第一个可玩场景。

## 中期：扩展制作流程

- BGM 和音效时间线、淡入淡出和场景音频过渡。
- 配音文件管理与对白绑定，保留人工审核和替换流程。
- 可插拔图片生成/分割提供商，生成结果继续进入候选审核生命周期。
- 更丰富的发布目标与离线作品导出。
- 项目级模板与可复用剧情结构。

## 长期：团队与部署

- 可选 PostgreSQL 部署配置和对象存储适配器。
- 多用户项目权限、评论和审阅；在此之前不实现实时协同编辑。
- 面向团队的操作审计、成本预算和模型调用配额。
- 插件或集成机制，保持故事领域契约和安全边界稳定。

## 明确不应提前做的事

- 在没有授权、预算、审计和回滚前扩大 Agent 自主写入权限。
- 把通用 shell、SQL 或任意文件系统暴露给创作 Agent。
- 为追求功能数量破坏章节、场景、镜头卡的主要写作路径。
- 在备份与迁移方案完成前更换数据库或素材存储结构。
- 把复杂真实背景的规则抠图描述成可靠的语义分割。

提出新方向时，请同时说明目标用户、真实工作流、现有方案为什么不够、数据与安全影响，以及如何验证完成。
