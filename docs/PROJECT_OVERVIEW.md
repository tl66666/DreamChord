# DreamChord 项目概览

这份说明面向 HR、老师、项目评审和第一次了解仓库的开发者。内容只描述当前仓库已经实现并经过测试的能力。

## 一句话说明

DreamChord 是一个本地运行的视觉小说创作平台，把长篇写作、分支剧情、角色舞台状态、素材处理、实时预览和可审核的创作 Agent 放进同一套工作流。

## 用户与场景

目标用户是个人视觉小说作者、剧情策划、独立游戏团队和希望把小说改造成互动故事的创作者。典型任务包括：

- 把正文拆成章节、场景和可播放的镜头。
- 维护不同分支中的人物、背景和选择结果。
- 上传角色图并准备成可复用立绘、背景或 CG。
- 检查剧情断路、人物设定和素材缺口。
- 让 AI 辅助续写或修改，同时保留作者最终决定权。

## 产品方案

DreamChord 采用“镜头卡写作、节点图检查”的组合。创作者不需要先手动画节点和连线；编辑器根据场景、镜头和选择生成图结构，节点图用于检查不可达内容、错误跳转和分支汇合。

舞台状态按剧情路径计算。一个场景后继续对话时，原背景和未退场人物保持不变；新增角色只产生登场变化，明确退场才隐藏人物。每条分支独立计算，汇合状态不一致时提示作者重置。

## Agent 的实际作用

Agent 与项目数据连接，但没有任意写入权限。它可以管理多个独立对话，读取当前项目、角色、素材、故事圣经、剧情图和分层记忆，并通过白名单工具检查问题或准备提案。

普通问题走自然对话；明确的剧情修改才进入受控行动。剧情补丁先校验和预览，作者确认后应用，并保留来源与撤销条件。这个设计解决了两类常见问题：通用聊天不知道项目上下文，以及 AI 直接改稿导致内容不可控。

实现上采用自研的 ReAct 风格执行循环，而不是直接引入一个“全自动改稿”的黑盒 Agent 框架：服务端先做本地意图路由，再按优先级装配项目上下文和记忆；模型只返回结构化 `tool_call` 或 `final`；工具输入经过 Zod schema 校验，剧情补丁再由共享的 `story-domain` 做图结构和版本校验。只有作者审批后，补丁才会进入正式剧情。当前模型层通过 OpenAI 兼容协议适配 GLM、DeepSeek、Kimi、OpenAI 或自定义 Provider；没有 API Key 时则由本地助手处理项目检索、创作知识、结构草案和只读公共知识查询。

## 技术实现要点

| 难点 | 当前实现 |
|---|---|
| 长篇剧情可维护性 | 项目、章节、场景、镜头卡和剧情图分层 |
| 分支舞台一致性 | 按路径继承背景与人物，分支隔离，汇合歧义提示 |
| 自动保存可靠性 | 串行保存、最新修订追赶、版本冲突保护 |
| Agent 可控性 | 对话路由、上下文装配、白名单工具、补丁校验、审批和撤销 |
| Agent 执行循环 | 自研 ReAct 风格 Observe → Reason → Act → Validate → Approve，最多限制工具步数 |
| 长期上下文 | 项目/对话作用域、六类记忆、建议/启用/遗忘状态和来源记录 |
| 用户图片可用性 | Sharp 内容检查、原图保留、候选衍生图和人工接受流程 |
| 项目可迁移性 | 带素材字节与哈希的版本化备份、严格导入和 ID 重映射 |
| 普通用户部署 | Windows 一键启动、环境诊断、数据备份和健康检查 |

## 工程结构

项目采用 pnpm monorepo：React/TypeScript 前端、Express/Prisma/SQLite 后端，以及前后端共享的剧情领域包。剧情规则不只存在于界面中，Agent、API 和播放器使用同一套图与补丁约束。

测试分为领域规则、服务端、前端组件和真实浏览器验收。发布前执行 lint、全量测试、生产构建和工作区就绪检查。

## 可以如何演示

建议使用官方演示账号 `demo / demo123`，按下面顺序展示：

1. 首页打开“第一根弦：消失的终页”。
2. 在故事编辑器展示镜头卡和舞台状态继承。
3. 打开流程图展示选择分支和汇合。
4. 在 Agent 中不选章节询问项目概况，再绑定章节提出剧情修改。
5. 在素材库上传白底人物图，生成候选立绘并绑定角色。
6. 完整预览故事，证明编辑数据能进入播放器。

真实界面截图保存在 `docs/screenshots/`，项目展示页为 `docs/showcase.html`。

## 当前边界

- 产品目前面向本地单用户，没有多人实时协作。
- 规则抠图适合透明或纯色背景，不等同于通用语义分割模型。
- 项目检索、创作知识解释、公共知识只读查询和本地结构草案不依赖模型；开放式长文创作仍需要作者配置外部模型服务。
- Agent 刻意不提供任意 shell、文件系统、SQL 或浏览器工具。
- 配音生成、完整 BGM 时间线和 PostgreSQL 部署配置属于后续方向。

这些边界在 [项目路线图](ROADMAP.md) 中单独维护，不会作为当前版本功能宣传。
