About

把节点图的复杂性隐藏在镜头卡背后

DreamChord 是一个面向非技术用户的视觉小说创作平台。用户只需像写小说一样填写背景、选择角色、编写台词，系统自动将其转换为可播放的节点图和运行时场景。无需学习 Ren'Py 脚本语言，无需理解节点连线，用"章节 → 场景 → 镜头卡"的方式组织剧情，一键发布即可在浏览器中播放。

痛点：节点图太难
镜头卡写作界面，用户只需填写"谁说了什么"，系统自动生成节点和连线
痛点：没有可视化预览
编辑器右栏实时预览，每改一个字、换一个背景立刻看到播放效果
痛点：分支管理混乱
自动检测分支去向，流程图可视化所有场景和分支连线，汇合点紫色高亮
痛点：AI 辅助缺失
编辑器内置 AI 助手，支持润色、续写、生成选项、分支回应、生成剧情图
痛点：无法即时分享
一键发布到作品广场，分享链接即可在浏览器中播放
痛点：长篇组织困难
章节 → 场景 → 镜头卡三级结构，每章独立画布，顶部标签切换
Interface

五大功能模块真实界面展示

以下均为项目实际运行截图，非设计稿。从编辑器到流程图，从 AI 写作到素材管理，每一处界面都围绕"让创作更简单"设计。

localhost:5173/editor
01 / Editor

镜头卡编辑器

三栏布局：左侧章节场景树管理剧情结构，中间镜头卡列表像写小说一样编写台词和旁白，右侧实时预览每一步修改的播放效果。无需理解节点图，系统自动生成所有节点和连线。

  • 章节 → 场景 → 镜头卡三级结构
  • 6 种镜头卡类型（对话/旁白/心理/回忆/系统/选项）
  • 右栏实时预览，所见即所得
  • 长文导入：粘贴小说正文自动拆分为镜头卡
localhost:5173/editor → 剧情流程图
02 / Flowchart

剧情流程图

galgame 风格的全局视图，将所有场景和分支连线可视化。普通连接用实线箭头，选项分支用虚线标注，汇合场景紫色高亮。支持拖拽调整场景位置和创建场景间连接。

  • 实线=普通连接，虚线=选项分支
  • 汇合场景紫色高亮 + GitMerge 徽章
  • 选项节点粉色边框 + 分支数量标识
  • 拖拽场景卡片调整位置，拖拽创建连接
localhost:5173/ai-writing
03 / AI Writing

AI 写作辅助台

五种写作模式覆盖从大纲到台词的全流程。选择任务类型、输入上下文，AI 生成结果可直接复制或保存到草稿箱。未配置 AI 时使用本地规则生成基础内容，配置后支持 GLM / DeepSeek / Kimi / OpenAI。

  • 整理大纲 · 场景续写 · 角色台词 · 选项分支 · 节点草稿
  • 本地辅助模式 + 云端 AI 模式双轨
  • 草稿箱持久化保存，随时复制到编辑器
  • BaseOpenAICompatibleProvider 统一接口
localhost:5173/assets
04 / Assets

素材库

角色立绘、背景图、表情差分统一管理。每个角色卡片包含立绘、一句话简介、人物小传、故事大纲和戏剧冲突（粉色高亮），底部表情差分选择器支持 normal / smile / serious 等状态切换。

  • 角色立绘 + 表情差分（normal / smile / serious）
  • 背景图库：樱花、教室、咖啡馆、星空
  • 戏剧冲突高亮策略，降低信息扫描成本
  • 支持自定义上传素材

Editor

六种镜头卡覆盖全部叙事场景

每张镜头卡编译为背景节点、角色节点和文本节点，系统自动生成连线。选项卡额外生成 choice-N 出边连接到分支场景。

💬
角色对话
角色之间的台词交流，支持表情和站位设置
→ dialogue 节点
📖
旁白
场景描述、时间流逝、环境变化等叙述文字
→ subtitle 节点
💭
心理描写
角色内心独白，保持角色身份但不显示立绘
→ dialogue 节点 (role 保持)
🌟
回忆镜头
闪回片段，过去事件的视觉化呈现
→ subtitle 节点
⚙
系统提示
系统幽灵发出的提示信息，Meta 叙事元素
→ subtitle 节点 (role=ghost)
🔀
选项分支
多选项分支点，每个选项可连接到独立场景
→ choice 节点 (choice-N 出边)
Workflow

从写作到播放的完整链路

用户的写作方式直接映射为运行时节点图，系统自动完成转换。

01

用户创建镜头卡

填写背景、选择角色、编写台词——像写小说一样

background → character → dialogue
02

系统自动生成节点和边

createSceneNodes 创建节点，chainEdges 连接节点，normalEdge 连接场景

nodes[] + edges[] → 节点图
03

转换器 DFS 遍历生成 RuntimeStory

convertFlowToRuntime 从起始节点深度优先遍历，追踪角色状态和背景

nodes + edges → RuntimeStory.scenes[]
04

运行时引擎驱动播放器

createRuntimeEngine 管理场景推进、分支选择和事件触发

next() → choose() → emit()
05

播放器渲染视觉效果

打字机效果、角色立绘、选项渲染、Web Audio 音效

VisualNovelPlayer → 浏览器

Quick Start

三步启动项目

Node.js 20+ 环境，pnpm 或 npm 均可。默认测试账号 demo / demo123。

01

安装依赖

克隆仓库后安装前后端依赖

git clone https://github.com/tl66666/DreamChord.git cd DreamChord && pnpm install
02

初始化数据库

配置 .env，运行 Prisma 迁移和种子

cd apps/server cp .env.example .env npx prisma migrate dev && npx prisma db seed
03

启动开发服务器

双击 start-dreamchord.bat 或手动启动前后端

cd apps/web && npm run dev cd apps/server && npm run dev
