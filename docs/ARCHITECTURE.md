# DreamChord 系统架构

更新日期：2026-07-12，对应版本：0.2.0。

本文面向准备阅读代码或继续开发的人，说明模块边界和关键数据流。实现级细节与安全约束继续维护在 [AI 与架构交接](AI_HANDOFF.md) 和根目录 [CLAUDE.md](../CLAUDE.md)。

## 总体结构

```text
React Web
  首页 / 故事编辑器 / 素材库 / Agent / 播放器
        |
        | REST + JWT
        v
Express API
  认证 / 项目 / 章节 / 素材 / Agent / 备份
        |
        +-> Prisma + SQLite
        +-> Sharp 图片处理
        +-> OpenAI 兼容模型接口
        +-> @dreamchord/story-domain
```

仓库使用 pnpm workspace：

| 路径 | 责任 |
|---|---|
| `apps/web` | 界面、编辑状态、API 客户端、播放器 |
| `apps/server` | HTTP API、授权、数据库、Agent 服务、媒体处理 |
| `packages/story-domain` | 共享剧情图、补丁协议、验证和健康规则 |
| `scripts` | 启动、诊断、数据库备份和就绪检查 |
| `docs` | 用户、产品、架构与发布文档 |

## 故事数据模型

创作层级是项目、章节、场景和镜头卡，持久化底层使用节点与边。编辑器把服务端记录转换成 React Flow 数据，播放器再把同一图结构转换成运行时事件。

共享剧情规则放在 `packages/story-domain`。前端、服务端和 Agent 不应各自发明不同的节点或补丁格式。

关键约束：

- 节点和边 ID 唯一，边只能引用存在的节点。
- 选项出口保留选项句柄与目标分支的对应关系。
- 保存请求携带正数 `baseVersion`。
- 服务端只在版本匹配时原子递增章节版本。
- 旧版本保存返回 `409`，不能覆盖新内容。
- 图转换必须确定性，同一输入得到同一运行结构。

## 编辑器状态与保存

`useEditorStore` 是当前章节图的单一状态源。用户操作通过快照进入撤销/重做历史，载入新章节时重置历史。

`SaveCoordinator` 串行处理自动保存：同一时间最多一个请求，保存过程中发生的新修改合并为下一次最新修订。冲突和错误状态保持为未保存，离开页面保护不会误判为已完成。

舞台状态通过剧情路径计算。背景和人物动作不是每张卡的完整截图，而是对之前状态的增量变化。分支分别计算；汇合得到多个不同状态时返回歧义结果，要求作者显式重置。

## 创作 Agent

Agent 有编辑器内紧凑面板和全屏多对话工作区两种界面。服务端核心位于 `apps/server/src/agent`，路由位于 `apps/server/src/routes/agent.ts`。

### 请求路由

```text
用户消息
  -> 本地即时：问候、时间、能力、下一步建议
  -> 只读对话：一般问题和项目分析
  -> 受控行动：明确的剧情修改或素材处理
```

只读对话可以读取项目，但不能应用补丁。受控行动需要足够作用域，并进入工具、验证、提案和审批流程。

### 上下文与记忆

每次运行从当前对话、滚动摘要、项目/章节/场景、故事圣经、角色、素材、剧情健康报告和分层记忆装配上下文。

六类记忆为 `canon`、`character`、`plot`、`decision`、`preference`、`artifact`；状态为 `suggested`、`active`、`forgotten`。先按用户、项目、对话和状态隔离，再按置顶、重要性和相关度排序。

### 工具与写入

Agent 只能使用注册的业务工具。模型没有数据库、shell、任意文件系统或通用网络写入权限。

```text
读取上下文 -> 制定计划 -> 调用白名单工具 -> 校验
-> 生成提案 -> 作者预览 -> 应用或拒绝 -> 必要时撤销
```

剧情应用记录来源和基础版本。只有章节仍处在应用后的版本时才能撤销，避免覆盖作者之后的编辑。

## 素材处理

上传文件先由 Sharp 解码检查，不能只信文件扩展名和浏览器 MIME。原始文件保留，衍生结果以 `proposed` 候选保存。

| 用途 | 输出 |
|---|---|
| 角色立绘 | 1024×1536 透明 PNG |
| CG | 1920×1080 WebP |
| 背景 | 1920×1080 WebP |

接受候选后才成为正式素材。立绘还需要在项目里绑定角色和表情。删除操作会检查角色、表情和故事引用，避免留下破图。

## 备份与恢复

项目导出使用版本化的 `dreamchord-project` 清单，携带项目数据、剧情图、故事圣经、角色、支持的记忆以及实际引用的上传文件。文件记录字节长度、MIME 和 SHA-256。

导入使用严格 Zod 校验和大小限制，创建新项目并重映射内部 ID，不把外部数据库 ID 直接写入现有项目，也不覆盖源项目。

## 安全边界

- JWT 保护用户资源，所有项目、对话、记忆和素材操作检查所有权。
- API Key 只用于当前模型请求，不写入消息或运行记录。
- 自定义模型地址在实际传输阶段重新检查，重定向逐跳验证。
- 静态媒体只允许已知扩展名，并返回 `X-Content-Type-Options: nosniff`。
- `.env`、SQLite、上传目录和日志不进入 Git。

更多说明见 [安全说明](../SECURITY.md)。

## 扩展新功能时

- 新剧情规则先进入共享领域包，再接入服务端和前端。
- 新 Agent 工具同时增加输入/输出 schema、注册、授权、资源限制和测试。
- 新媒体处理先生成候选，不绕过接受流程。
- 数据库 schema 变化必须考虑旧数据备份、迁移和一键启动。
- 行为修改先写回归测试，再运行全量验证。

推荐命令：

```bash
pnpm lint
pnpm test
pnpm build
pnpm test:readiness
git diff --check
```
