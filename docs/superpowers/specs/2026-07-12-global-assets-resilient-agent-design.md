# DreamChord 全局素材库与韧性 Agent 设计

## 目标

修复 Agent 任务因工具参数格式错误而整体失败的问题；让对话可以只绑定项目、也可以进一步绑定章节；把上传素材从“某个故事的附件”升级为“用户自己的素材库”，使同一素材能在任意故事章节中使用；未配置 API Key 时仍提供明确、可用的本地助手能力。

## 方案选择

### 采用：用户级素材所有权 + 可选来源项目

素材记录增加明确的 `ownerId`，`projectId` 改为可空，仅保留来源或导入关系。素材库默认查询当前用户的全部素材，上传不要求选择故事。编辑器仍传入当前项目作为“使用目标”，角色立绘接受时绑定到该项目角色；背景和 CG 通过 URL 可直接进入任意项目镜头。

相比“跨项目查询现有项目素材”，该方案不会在删除来源项目时级联删除其他故事正在使用的文件，也允许用户在还没决定故事前先整理素材。相比新建隐藏素材项目，它没有伪项目和权限绕行。

## Agent 工具韧性

模型响应协议与具体工具输入是两层校验。协议合法但工具输入错误时，执行器不立刻终止任务，而是：

1. 记录 `tool_input_repair` 时间线事件；
2. 把工具名、期望字段和简化错误返回给模型；
3. 要求模型重新发送单个合法 `tool_call`；
4. 最多修复两次，之后返回中文的稳定错误码，不暴露原始 Zod JSON。

对高频兼容输入做安全归一化：单素材工具收到唯一的 `ids: [id]` 时转为 `assetId: id`；多个 ID 不擅自选择，要求模型逐个调用。系统提示明确 `inspect_asset`、`prepare_*` 都只接受一个 `assetId`。

## 项目级与章节级对话

- 项目始终必选。
- 章节选择器增加“不绑定章节（项目对话）”。
- 未选章节时，会话 scope 为 `project`，可以读取项目简介、角色、共享素材、故事圣经、对话历史和记忆，也可以进行本地项目摘要与素材盘点。
- 镜头、场景、章节工具在没有章节时不可用，并返回“请选择章节后再修改剧情”的可行动提示。
- 只有选定章节且补丁通过验证时，Agent 才能生成待审批的剧情修改。
- 切换章节不会删除或混淆对话；对话可通过设置更新其章节绑定。

## 无 API Key 的本地助手模式

本地模式不是伪装成生成模型，而是确定性能力：

- 剧情体检：运行共享 story-domain 规则，列出断路、空选项和不可达节点；
- 项目摘要：章节、角色、素材、记忆和故事圣经概览；
- 素材盘点：按类型统计并列出近期素材，给出透明图、白底图和复杂背景的处理建议；
- 角色盘点：列出角色、默认立绘和缺失项；
- 普通聊天：回答当前可执行能力，并根据关键词路由到上述分析。

本地模式会把用户消息和助手结果持久化到正常对话中。它不生成文学文本、不调用外网、不创建剧情补丁。涉及续写、润色或分支生成时，回复中明确说明需要配置模型，并提供设置入口；任务状态为完成而不是失败。

## 全局素材生命周期

### 上传与浏览

- 素材库不再显示故事选择器。
- 上传请求只包含文件和类型，后端以认证用户作为 owner。
- 支持按类型筛选、搜索、重命名、替换、处理和删除。
- 编辑器素材面板查询同一用户的全局素材，并可在任意项目中选择。

### 图片处理与角色绑定

- 原图和衍生图都归当前用户。
- 在全局素材库处理立绘时，可以先接受为透明 PNG 素材，不强制创建项目角色。
- 从故事编辑器处理并接受立绘时，传入当前项目 ID 和角色名，立即绑定为该项目角色/表情。
- CG 和背景接受后成为全局素材，可在任意故事镜头中选择。

### 删除与备份

- 删除前检查该用户所有项目的封面、角色、表情、节点和故事圣经引用；仍被使用时拒绝删除并提示先解除引用。
- 删除来源项目不会删除用户级素材。
- 导出项目时除来源素材外，还扫描项目封面、角色和节点 URL，把实际引用到的全局素材一并嵌入备份。
- 导入素材归导入用户所有，同时记录导入项目作为来源。

## 错误呈现

- 时间线区分“格式修复”“工具参数修复”“工具执行”和“任务完成”。
- 用户界面不直接展示 Zod issues 数组、堆栈或密钥相关内容。
- 提供商认证、余额、网络和超时错误转换为简短中文说明，并保留可重试入口。
- 本地能力可以完成的请求不因缺少 API Key 失败。

## 测试与验收

- 执行器测试覆盖单 `ids` 自动归一化、多个 `ids` 请求修复、修复后继续执行和友好失败。
- 运行服务测试覆盖无章节项目对话、本地助手完成、无章节补丁被拒绝。
- 素材路由与服务测试覆盖用户级查询、无项目上传、跨项目可见、来源项目删除后保留、跨项目引用删除保护。
- 项目备份测试覆盖引用的全局素材被嵌入。
- 前端测试覆盖“不绑定章节”、项目级新对话、无 Key 本地运行、共享素材库无项目选择器和立绘目标项目绑定。
- 浏览器验收覆盖截图中的失败路径、无 Key 对话、选章节续写入口、全局素材上传与两个项目复用。
