# DreamChord 零配置 Agent 与展示页恢复设计

更新日期：2026-07-12

## 问题

无 API Key 时，本地助手只识别少量正则意图。未命中的问题统一返回“不能可靠回答”，已经加载的对话历史和项目记忆也没有传入本地助手。结果是时间和项目盘点可用，但创作知识、追问、记忆查询和一般事实问题几乎不可用。

展示页后续迭代连续扩写了原页面模块和文案，偏离了已有视觉与叙事节奏。新增截图中包含移动宽度证据，但 DreamChord 的主要创作体验是桌面 Web 工作台，项目展示应优先使用桌面真实截图。

## 零配置 Agent

本地 Agent 使用三层能力，按顺序匹配：

1. 项目与对话：回答项目、章节、人物、素材、剧情健康、最近对话和启用记忆。
2. 创作知识：从内置视觉小说知识库检索术语和方法，返回定义、适用场景和在 DreamChord 中的操作建议。
3. 公共知识：对“X 是什么”“谁是 X”“介绍 X”等事实问题，调用严格限制为中文 Wikipedia 的只读查询，返回摘要和来源。

开放式续写在无模型时不再只拒绝。系统根据项目简介、故事圣经、角色和章节生成可继续编辑的三步剧情方案与短示例，但明确标记为“本地结构草案”，不伪装成模型生成，也不直接写入章节。

公共知识工具只发送去除问句词后的查询关键词，不发送项目、对话、记忆或 API Key。只允许 `https://zh.wikipedia.org/`，设置超时、响应长度限制和错误降级。网络不可用时继续提供项目能力与创作知识入口。

## 上下文接入

`runService` 把已装配的 `conversation-history`、`conversation-summary` 和 `memory` 来源传入本地助手。它们只用于总结当前对话和列出启用记忆，不跨项目或跨对话重新查询。

本地助手改为异步接口，以便公共知识查询。外部模型分支和审批协议保持不变。
