# DreamChord 启动、中文文档与编辑器素材闭环设计

## 目标

让普通 Windows 用户下载项目后可以可靠地一键启动，用中文理解完整使用流程，并把自己上传的图片经过人工或 Agent 辅助处理后，直接用于故事场景、CG 和角色立绘。

本次不重写已经完成的 Agent、分层记忆、图像分析和候选素材审核机制。改动聚焦于用户目前实际遇到的断点：重复启动时 Prisma DLL 被占用、交付文档语言不一致、处理后的素材没有形成清晰的编辑器使用闭环。

## 方案选择

### 方案 A：状态感知启动器（采用）

启动器先识别当前机器上是否已有健康的 DreamChord 后端和前端。正常启动时，若已有完整实例，则直接打开已有页面并成功退出；若只有后端运行，则提示用户使用现有服务或关闭旧服务后重新启动。只有在没有 DreamChord 服务占用 Prisma Client 时，才安装依赖、生成 Prisma Client、同步数据库并启动新实例。

优点是直接解决 Windows 文件锁根因，不会强制结束用户进程，也不会把其他占用相同端口的程序误认成 DreamChord。缺点是启动器需要增加健康检查和少量状态判断。

### 方案 B：每次强制结束旧进程

实现较短，但可能误杀用户正在使用的服务，丢失尚未保存的数据，也违反项目现有的“不使用 Stop-Process”约束，因此不采用。

### 方案 C：跳过 Prisma generate

可绕过当前报错，但新下载、依赖升级或 schema 变化时会留下旧客户端，后续以更隐蔽的方式失败，因此不采用。

## 启动器设计

### 实例识别

- 扫描后端候选端口 `3001..3010`，请求 `/api/health`。
- 只有返回成功且 JSON 中 `service` 明确等于 `dreamchord-server` 才算 DreamChord 后端，不能只凭端口占用判断。
- 扫描前端候选端口 `5173..5183`，请求页面并验证 DreamChord 页面标识。
- 识别过程中网络错误、超时和非 DreamChord 响应只作为“未找到”，不终止启动器。

### 正常启动

- 已发现健康后端和前端：输出中文状态，打开已有前端，退出码为 0，不运行安装、备份、Prisma 或 seed。
- 只发现健康后端：输出中文说明，阻止执行 `prisma generate`，避免 DLL 锁错误；提示关闭旧 DreamChord 服务窗口后重试。
- 未发现健康实例：执行环境检查、固定版本依赖安装、数据库备份、Prisma Client 生成、schema 同步、幂等 seed，再启动前后端。

### 安装检查模式

`-SetupOnly` 需要执行安装与数据库检查。如果发现运行中的 DreamChord 后端，则明确拒绝执行并说明原因，避免在 DLL 被占用时尝试升级。

### 错误与诊断

- `.bat`、PowerShell 主脚本和 `doctor.ps1` 的用户提示全部使用中文。
- 保留 `[PASS]`、`[WARN]`、`[FAIL]` 标记，便于机器检查和用户截图反馈。
- doctor 区分“端口被其他程序占用”和“检测到 DreamChord 正在运行”。
- 启动器不自动结束任何进程。

## 中文交付文档

根 README 改为自然中文，并作为新用户唯一入口，覆盖：

- 产品定位与已完成能力；
- Windows 一键启动、首次安装要求、重复启动行为；
- 演示账号与官方示例；
- 从章节、场景、镜头卡到完整预览的创作步骤；
- 上传素材、图像检查、处理候选、接受或拒绝、放入故事的完整路径；
- 透明 PNG、白色或浅色纯背景、复杂背景图片各自的适用性；
- 手动操作和 Agent 工具调用的区别；
- 本地数据、上传文件、备份和隐私说明；
- Prisma DLL 锁等常见问题；
- 开发命令、目录结构与 GitHub Pages 展示页。

不夸大图像能力：透明图最稳定；白色或浅色纯背景可用边缘连通消除；复杂背景只能生成建议和候选，不承诺语义级精准抠图。

## 编辑器素材闭环

### 场景与 CG

- 故事卡的“背景”入口同时展示 `BACKGROUND` 和 `CG` 类型。
- 选择背景素材后写入当前镜头卡的 `background`，沿用现有 graph 转换、保存和播放器渲染路径。
- CG 作为全屏画面使用，与背景采用相同的故事渲染语义，不新增无用的节点类型。
- 新建场景使用已有内置教室背景作为默认值，避免空 URL 导致预览破损；用户可立即替换为项目素材。

### 角色立绘

- 上传角色图后，图像处理生成候选透明 PNG；只有用户接受并绑定有效角色名后，才写入项目角色。
- 接受后立即刷新 `project.characters`，当前镜头卡的角色、说话人和表情选择器无需刷新页面即可使用。
- 角色图不通过通用 CG URL 直接塞入角色节点，继续使用已有项目角色绑定模型，保证人物名称、表情和立绘路径一致。

### 素材面板交互

- 从镜头卡打开素材面板时，明确显示“背景 / CG”用途。
- 处理并接受背景或 CG 后刷新素材列表，用户可以立即选入当前镜头。
- 处理并接受角色立绘后显示成功反馈，并立即加入角色选择器。
- 原图始终保留；派生图先进入候选状态，接受后才成为可用素材，拒绝或删除通过服务层清理。

## 数据流

```text
上传原图
  -> 服务端解码与图像检查
  -> 推荐用途与处理配方
  -> 用户手动处理或 Agent 调用受限工具
  -> 生成 proposed 候选
  -> 用户预览并接受
  -> 背景/CG：进入项目素材列表 -> 写入镜头卡 background
  -> 立绘：绑定项目角色 -> 进入角色/说话人/表情选择器
  -> 自动保存章节 -> 完整预览和播放器渲染
```

## 测试与验收

### 自动化

- 启动器回归测试确保实例检查发生在 `prisma generate` 之前。
- 测试正常启动复用健康实例、安装检查拒绝被占用的 Prisma Client、中文提示和禁止 `Stop-Process`。
- 组件测试确保背景选择器同时显示 BACKGROUND 与 CG。
- 编辑器测试确保新场景有有效默认背景，接受角色立绘后项目角色立即可选。
- 保持现有图像检查、处理、Agent 工具、保存和播放器测试通过。

### 浏览器验收

- 桌面和移动尺寸打开故事工作台，无控件遮挡和控制台错误。
- 上传白底人物图，检查建议，生成透明立绘，接受并绑定角色，在镜头卡中选用并预览。
- 上传场景图，处理为背景或 CG，接受并选入镜头卡，保存后完整预览可见。
- 新建场景默认画面可见，场景、对话、选项、分支与汇合流程仍可用。

### 全量门禁

运行 `pnpm lint`、`pnpm test`、`pnpm build`、`pnpm test:readiness`、`git diff --check`，并在提交前核对敏感文件和生成物未进入 Git。
