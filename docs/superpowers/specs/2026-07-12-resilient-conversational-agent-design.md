# DreamChord 自然对话与弹性 Agent 设计

## 问题

当前外部模型必须返回严格的 Agent JSON。GLM 对简单问候返回了带 `patch: null` 的结果，协议校验连续两次失败，最终把“你好”显示为任务失败。无 Key 本地助手也会把未识别问题统一回答成项目概况，缺少自然对话感。

## 目标

- “你好”“你能做什么”“谢谢”等普通对话立即得到自然回复。
- 项目概况、角色、素材、剧情体检继续使用确定性本地能力。
- 外部模型返回 `patch: null`、代码块 JSON、JSON 前后说明时能够安全解析。
- 模型始终无法绕过工具白名单和变更审批。
- 即使模型最终没有遵守结构化协议，也把可读文本保存为只读答复，不再把普通问题标成失败。
- 运行记录使用用户能理解的中文，不展示 `format_repair` 等内部事件名。

## 方案

Agent 分为三层：

1. **即时对话层**：本地识别问候、致谢、身份与能力问题，结合当前项目给出简短回复，不消耗 API。
2. **项目理解层**：本地完成项目概况、角色与素材盘点、剧情结构检查。外部模型可结合对话历史、分层记忆和项目上下文回答更开放的问题。
3. **工具执行层**：只有结构化 `tool_call` 才能读取工具或生成剧情/素材候选。所有写入继续等待作者审批。

## 协议容错

- `patch: null` 等同于没有补丁。
- `plan: null` 或缺失时使用空数组。
- 支持完整 Markdown JSON 代码块。
- 支持从简短前后说明中提取唯一 JSON 对象。
- 首次格式异常仍要求模型修复。
- 达到修复上限时，若响应包含可读自然语言，则生成只读完成结果并记录“已转为安全对话答复”；不执行工具、不产生补丁。
- 空响应、只有损坏 JSON 或安全上无法解释的响应仍失败，并显示明确原因。

## 本地助手

本地助手新增：

- 问候与致谢；
- “你是谁 / 能做什么 / 怎么使用”能力说明；
- 当前项目的下一步建议；
- 项目、人物、素材、剧情健康查询；
- 无 Key 创作请求的配置引导。

回复应点明当前项目，但不机械重复统计信息。

## 界面

- 空白对话提供四个可点击起始任务：了解项目、检查剧情、梳理角色、素材建议。
- 运行按钮统一为“发送给 Agent”，不再让普通聊天看起来像批处理任务。
- 时间线把修复事件显示为“正在适配模型返回格式”“已转为安全对话答复”。
- 失败状态使用警告图标和错误说明；完成状态使用成功图标。
- 项目展示页和 README 重点说明自然对话、上下文、记忆、工具、审批与格式容错，并使用本轮真实截图。

## 测试与验收

- 协议测试覆盖 `patch: null`、代码块、前后说明和不可恢复输入。
- 执行器测试覆盖一次修复后成功、修复耗尽后自然文本降级、空响应仍失败。
- 本地助手测试覆盖问候、能力问题、项目下一步和原有查询。
- 前端测试覆盖快捷起始任务和中文时间线。
- 使用真实 GLM 配置复测“你好”，确认生成助手消息且运行状态为完成。
- 桌面与 390px 移动端截图无重叠，浏览器无异常请求。

## 安全边界

- 自然文本降级永远不产生补丁、记忆建议或素材产物。
- 工具名和参数仍经过现有白名单与 Zod 校验。
- 项目级对话不能直接修改章节。
- API Key 不写入消息、运行记录、截图或项目备份。
