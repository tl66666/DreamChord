# DreamChord 创作 Agent 使用指南

DreamChord Agent 同时承担对话、项目理解和受控执行三种工作。普通聊天不需要先选章节，也不需要把问题写成命令。

## 从哪里开始

进入“创作 Agent”后先选择故事项目。新对话默认是“项目对话”，可以直接问：

- 你好，你能做什么？
- 概括整个项目，并告诉我下一步先完善什么。
- 梳理现有角色的目标、关系和缺失设定。
- 盘点素材库，还缺哪些背景、CG 和立绘？
- 检查整部故事的断路、死路和薄弱分支。

页面中的“了解项目、检查剧情、梳理角色、素材建议”会把常用问题填入输入框，作者仍可在发送前修改。

## 什么时候绑定章节

讨论世界观、人物关系、整体结构和素材时不用绑定章节。需要润色镜头、续写场景、补充分支或生成剧情补丁时，先在顶部选择章节，再选择作用范围：当前镜头、当前场景、当前章节或全项目。

项目对话是只读范围。即使模型错误地返回剧情补丁，服务端也会拒绝写入，并提示先选择章节。

## Agent 读取什么

每次运行按优先级组合：

1. 当前问题、章节和选中范围；
2. 故事圣经与固定约束；
3. 当前对话最近消息和滚动摘要；
4. 项目记忆、角色、全局素材和剧情健康报告。

六类记忆分别是世界事实、角色、剧情线、创作决定、作者偏好和已应用产物。建议记忆不会自动成为正式设定，作者确认后才进入后续上下文。

## 工具和审批

Agent 只能调用 DreamChord 注册的工具，例如读取章节、检查剧情图、搜索记忆、查询角色素材、检查图片，以及生成立绘、CG 或背景候选。它不能调用 shell、任意文件系统或数据库。

剧情修改先生成差异预览，作者选择应用或放弃。图片处理先生成候选，作者检查透明边缘和画面后再接受。已应用剧情可以在版本仍匹配时撤销。

## 没有 API Key 时

本地助手可以完成：

- 问候、能力说明和项目下一步建议；
- 项目、角色与素材盘点；
- 白底图和透明 PNG 导入建议；
- 确定性的剧情结构检查。

续写、润色、开放式改写和新剧情生成需要外部模型。可在设置页配置 GLM、DeepSeek、Kimi 或 OpenAI 兼容接口。

## 模型格式异常时

不同模型可能返回 `patch: null`、Markdown 代码块、JSON 前后说明或字段别名。DreamChord 会先规范化常见格式，再请求模型修复。仍然无法得到结构化结果但存在可读文本时，系统只把它保存为普通答复，不执行工具、不生成补丁，也不会把简单问题显示成任务失败。

空响应、损坏 JSON 和无法解释的结果仍会失败，界面会显示具体原因。重新发送问题不会覆盖原有故事内容。

## 图片素材任务

Agent 可以盘点账号全局素材并准备图片候选。透明 PNG 最适合角色立绘；白色或浅色平底可使用边缘连通去底。复杂真实背景不适合本地规则抠图，应先使用专业分割工具得到透明 PNG。

接受到全局素材库不等于绑定角色。进入目标故事后再选择角色和表情，素材便可用于镜头卡、实时预览和播放器。
