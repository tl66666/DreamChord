# DreamChord 创作 Agent 使用指南

DreamChord Agent 同时承担对话、项目理解和受控执行三种工作。普通聊天不需要先选章节，也不需要把问题写成命令。

## 从哪里开始

进入“创作 Agent”后先选择故事项目。新对话默认是“项目对话”，可以直接问：

- 你好，你能做什么？
- 现在几点？
- 蒙太奇和普通转场有什么区别？
- 概括整个项目，并告诉我下一步先完善什么。
- 梳理现有角色的目标、关系和缺失设定。
- 盘点素材库，还缺哪些背景、CG 和立绘？
- 检查整部故事的断路、死路和薄弱分支。

页面中的“了解项目、检查剧情、梳理角色、素材建议”会把常用问题填入输入框，作者仍可在发送前修改。

## 什么时候绑定章节

讨论世界观、人物关系、整体结构和素材时不用绑定章节。需要润色镜头、续写场景、补充分支或生成剧情补丁时，先在顶部选择章节，再选择作用范围：当前镜头、当前场景、当前章节或全项目。

项目对话默认是只读范围。没有绑定章节时仍然可以让 Agent 给出续写文字、人物小传或情节构思，但不会形成可应用补丁。即使模型错误地返回剧情补丁，服务端也会丢弃补丁、保留文字回答，不会把整次对话标记为失败。

## 三种执行方式

1. **本地即时回答**：问候、时间日期、能力说明和下一步建议，不调用外部模型。
2. **只读对话**：一般问题、创作讨论和项目分析。可以调用读取类工具，不能写剧情或生成素材候选。
3. **受控行动**：绑定章节后的明确改写，或明确的素材处理。工具产物必须经过作者预览和确认。

默认永远是只读。只有问题同时表现出明确修改意图，并且具备所需上下文时，才进入行动模式。

## Agent 读取什么

每次运行按优先级组合：

1. 当前问题、章节和选中范围；
2. 故事圣经与固定约束；
3. 当前对话最近消息和滚动摘要；
4. 项目记忆、角色、全局素材和剧情健康报告。

六类记忆分别是世界事实、角色、剧情线、创作决定、作者偏好和已应用产物。建议记忆不会自动成为正式设定，作者确认后才进入后续上下文。

## 工具和审批

Agent 只能调用 DreamChord 注册的工具，例如读取章节、检查剧情图、搜索记忆、查询角色素材、检查图片，以及生成立绘、CG 或背景候选。它不能调用 shell、任意文件系统或数据库。

剧情修改先生成差异预览，作者选择应用或放弃。图片处理先生成候选，作者检查透明边缘和画面后再接受。已应用剧情可以在版本仍匹配时撤销。

### 外部访问边界

内部 Tool Calling 是服务端的业务白名单，不是 MCP 或通用插件。只有两类外部网络请求：配置 API Key 后访问用户选择的 OpenAI 兼容模型接口；以及回答公共事实问题时只读访问中文 Wikipedia 摘要接口。Wikipedia 只接收清洗后的关键词，不接收项目内容、对话、记忆或密钥。Agent 不具备浏览器自动化、Shell、任意 HTTP、SQL 或文件系统工具；图片检查和白底去除由本机 Sharp 流水线完成。

完整清单、验证记录和后续方向见 [当前迭代交接](CURRENT_HANDOFF.md)。

## 没有 API Key 时

本地 Agent 仍然可以完成：

- 问候、能力说明、项目下一步建议和章节清单；
- 项目、角色、全局素材、当前对话和启用记忆盘点；
- 白底图、透明 PNG、立绘、CG 和背景导入建议；
- 蒙太奇、伏笔、悬念、节奏、人物弧光、分支设计等视觉小说知识解释；
- “X 是什么”“谁是 X”等常见事实的中文 Wikipedia 只读摘要查询；
- 根据当前项目上下文生成确定性续写草稿，并在作者确认前保持为未写入状态。

本地知识和公共查询不需要模型 Key，但公共查询需要网络。本地 Agent 还能根据项目上下文生成确定性续写草稿，并把用户或 Agent 已有的正文解析成场景、旁白、角色台词和选项，生成可审核补丁；只有项目中精确匹配的背景和角色才会自动绑定。续写成更自然的长文、对白润色和开放式改写建议在设置页配置 GLM、DeepSeek、Kimi 或 OpenAI 兼容接口。无论是否配置模型，修改故事都必须经过提案、校验和作者确认。

## 将草稿变成工作台场景

1. 在对话中生成或粘贴正文，选择需要保留的那一份草稿。
2. 点击“生成工作台场景”，并绑定目标章节。
3. Agent 会移除“续写草稿”“镜头卡说明”等非正文，按 `镜头`、`地点`、`旁白`、`角色`、`选项` 解析为场景与镜头卡。
4. 在补丁预览中检查场景标题、台词和素材匹配；未匹配的背景或立绘会保持为空，不会被默认素材替代。
5. 确认应用后，场景进入编辑器，可继续修改、补充素材、连接选项分支，或让 Agent 继续基于当前场景续写。

## 模型格式异常时

普通对话允许模型直接返回自然文本，第一条可读回答会立即保存，不需要反复修复 JSON。只有需要工具或审批的行动任务才要求结构化协议。不同模型在行动模式中可能返回 `patch: null`、Markdown 代码块、JSON 前后说明或字段别名，DreamChord 会规范化常见格式并在必要时请求修复。

空响应、网络错误和无法解释的结果仍可能失败，但失败状态与解释消息会在同一事务写入对话，用户不会只看到自己的问题。重新发送不会覆盖原有故事内容。

## 图片素材任务

Agent 可以盘点账号全局素材并准备图片候选。透明 PNG 最适合角色立绘；白色或浅色平底可使用边缘连通去底。复杂真实背景不适合本地规则抠图，应先使用专业分割工具得到透明 PNG。

接受到全局素材库不等于绑定角色。进入目标故事后再选择角色和表情，素材便可用于镜头卡、实时预览和播放器。
