# DreamChord 长篇创作与小说导入工作流

更新日期：2026-07-02

这份文档给后续开发者和 AI 使用，说明 DreamChord 不应该把所有剧情都堆成一条无限下拉列表，而应该按“章节、场景、镜头卡、节点图”的层级组织。

## 产品判断

DreamChord 面向的是不懂节点图的创作者。用户真正需要的不是先学会连线，而是：

1. 像写小说一样先把剧情写出来。
2. 像做分镜一样把剧情拆成一幕一幕。
3. 在每一幕里选择背景、登场角色、立绘状态、文本和选项。
4. 系统自动把这些镜头卡转换成视觉小说运行时需要的节点和连线。

节点图仍然重要，但它主要用于检查分支结构、跳转关系和汇合点，不应该成为长篇剧情的主要书写入口。

## 长篇剧情的层级

推荐结构：

| 层级 | 用途 | 编辑入口 |
|---|---|---|
| 项目 | 一部作品 | 首页 / 项目列表 |
| 章节 | 第一章、第二章、番外、结局线 | 编辑器顶部章节切换 |
| 场景 | `1-1`、`1-2`、`2-1` 这种具体幕次 | 左侧章节/场景树 |
| 镜头卡 | 背景、角色、表情、旁白、心理、对话、选项 | 中间故事工作台 |
| 节点图 | 检查分支、跳转、汇合 | 节点图视图 |

长篇写作时，用户应该在每一章自己的画布里写这一章。第一章结束后，新建第二章，再把章节顺序串起来。这样后续有几十个场景时，编辑区仍然可读。

## 当前已实现的编辑优化

`apps/web/src/editor/ShotCardEditor.tsx` 已加入：

- `导入长文`：把小说正文快速拆成镜头卡。
- `紧凑视图`：剧情很长时减少卡片高度，方便快速浏览。
- 镜头卡继续支持背景、多个角色、立绘状态、站位、说话人、心理描写、回忆、系统提示和选项分支。

长文导入规则：

- `角色：台词` 会识别成角色对话。
- 普通段落会识别成旁白。
- 包含“回忆、想起、曾经、小时候”等词，会标记为回忆镜头。
- 包含“心想、心里、暗想、我想”等词，会标记为心理描写。
- 包含“系统、提示、警告、权限、错误”等词，会标记为系统提示。
- 超长段落会按中文标点拆成多张镜头卡，避免一张卡文本过长。

## 长篇小说转视觉小说的推荐流程

1. 先在素材库准备角色和场景。
2. 新建项目，创建第一章。
3. 在左侧新建一个场景，例如 `1-1 开场`。
4. 点击故事工作台里的 `导入长文`。
5. 粘贴一段小说正文，不要一次导入整本书。建议每次导入一个场景或一小节。
6. 导入后逐张检查镜头卡，为关键卡补背景、角色、表情和站位。
7. 遇到玩家选择时，手动添加选项卡，并为每个选项创建或选择后续场景。
8. 一章写完后打开节点图，只检查分支是否断开，不在节点图里重写剧情。
9. 点击项目体检，修复空文本、断开的选项出口、孤立节点和不可达场景。
10. 保存并预览。

## 未来建议

下一轮优先做这些：

1. 导入前预览：长文导入后先显示“将生成哪些镜头卡”，用户确认后再写入。
2. 按章节导入：粘贴完整章节时，自动按 `###`、`第X章`、空行标题拆成多个场景。
3. 角色名映射：如果正文里出现“少女：”，允许用户映射到素材库里的“雪”。
4. 批量设置背景：导入后可以一键给当前场景所有镜头卡套同一个背景。
5. 分支摘要：节点图里只展示场景级分支，点击场景后再进入该场景内部镜头卡，避免大作品节点图过密。
6. AI 辅助拆稿：AI 根据小说段落自动建议场景切分、角色登场、情绪表情和选项位置。
