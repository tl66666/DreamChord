# DreamChord 创作者工作流

这份文档说明故事编辑器、素材库和创作 Agent 应该怎样配合。目标不是一次生成整部作品，而是让每次修改都能看懂、预览、保存和回退。

## 先建立作品骨架

一部作品对应一个项目。项目里按章节拆分长篇内容，章节里按地点、时间或叙事目的拆分场景，场景里用镜头卡表达实际播放内容。

```text
项目《第一根弦》
  第一章 消失的终页
    1-1 旧校舍门口
      背景建立 -> 雪登场 -> 对话 -> 影登场
    1-2 封锁区
      旁白 -> 选项 -> 两条调查分支
  第二章 回声
    2-1 档案室
```

章节适合独立保存和加载；场景适合表达一次连续的舞台；镜头卡适合控制一句话、一次人物变化或一个玩家选择。不要把整章塞进一个场景，也不要为了每句对话都新建场景。

## 舞台连续性

每个镜头的画面由之前沿当前路径发生的变化累积而成。

- 第一张卡建立背景并让人物登场。
- 后续对话默认保持当前背景和在场人物。
- 新人物出现时添加“登场 / 更新”，原有人物继续保持。
- 人物换表情或站位时使用“登场 / 更新”，不是先退场再登场。
- 人物真正离开画面时使用“退场”。
- 切换地点时更新背景，并明确处理还应不应该留在画面中的人物。

分支从各自入口继承状态，互不污染。两条分支汇合时，如果背景或人物不同，编辑器会把结果标记为不确定；在汇合后的第一张卡明确重设背景和人物即可。

## 镜头卡怎么拆

一张卡只表达一个主要节拍，通常是：

- 一句人物对话；
- 一段长度适中的旁白或心理；
- 一次背景、人物、表情或站位变化；
- 一组玩家选项；
- 一次跳转或场景结束。

文字太长时拆成多张卡。需要快速导入小说正文时，每次导入一个场景或一小节，先预览解析结果，再确认写入。

## 素材准备与复用

素材先进入账号的全局素材库，再被具体项目引用。上传时不需要指定章节。

### 角色立绘

1. 上传透明 PNG，或白色、浅色、颜色基本一致的平底人物图。
2. 运行图片检查，确认尺寸、透明度和推荐用途。
3. 生成透明立绘候选，检查头发、衣服边缘和半透明区域。
4. 接受到素材库。
5. 在目标项目中把素材绑定到角色和表情。
6. 在镜头卡选择角色、表情、站位和舞台动作。

透明 PNG 最稳定。复杂房间、街道、树林或渐变光影不适合本地规则去底，应先用专业分割工具处理。

### 背景与 CG

背景描述持续的场景空间；CG 是占据主要画面的剧情插图。两者都可以生成 1920×1080 WebP 候选，接受后从镜头卡的背景/CG 入口选择。

原图不会被处理结果覆盖。候选图拒绝后不会进入正式素材列表。

## Agent 应该什么时候使用

Agent 适合帮助作者读项目、查缺口和准备修改，不适合替代作者确认。

不绑定章节时可以：

- 讨论世界观、主题和人物关系；
- 概括整个项目；
- 检查角色、素材和剧情结构；
- 为下一章提供文字构思；
- 管理不同目标的独立对话。

绑定章节后可以：

- 续写当前场景；
- 润色对白或旁白；
- 补充分支；
- 生成结构化剧情补丁；
- 读取当前章节和选中范围进行定向修改。

应用前检查差异。重要设定放进故事圣经或启用的项目记忆，不要只留在聊天记录里。不同任务使用不同对话，例如“主线续写”“人物口吻”“支线检查”和“素材准备”。

## 每章完成检查

1. 从章节入口完整试玩所有主要路线。
2. 在流程图检查不可达场景、断开的选项和错误跳转。
3. 检查分支汇合后的背景和在场人物。
4. 检查说话人是否有对应角色与立绘。
5. 检查素材是否仍存在、是否有未接受候选。
6. 确认自动保存完成，没有冲突或失败状态。
7. 导出一次项目备份，验证重要素材已包含。

长篇拆分细节见 [长篇创作与小说导入](LONG_STORY_WORKFLOW.md)，Agent 操作见 [创作 Agent 使用指南](AGENT_GUIDE.md)。
