# DreamChord 新手入门

这份指南从下载项目开始，带你完成登录、打开故事、新建场景、添加对话和预览。全程不需要先学习节点图，也不需要配置模型 API Key。

## 运行前准备

- Windows 10 或 Windows 11。
- Node.js 20 或更高版本，建议使用 LTS 版本。
- 首次安装依赖时可以访问网络。
- 至少保留约 2 GB 可用磁盘空间，方便安装依赖、保存素材和数据库备份。

请下载或克隆完整仓库。只下载 `start-dreamchord.bat` 无法运行，因为启动器还需要项目代码、锁文件和脚本。

## 最简单的启动方式

1. 双击根目录的 `start-dreamchord.bat`。
2. 第一次启动会检查 Node.js、启用固定版本的 pnpm、安装依赖、准备数据库和演示数据。
3. 等待命令窗口显示前端和后端均已就绪。
4. 浏览器会自动打开；如果没有打开，手动访问命令窗口显示的前端地址。
5. 使用 `demo / demo123` 登录。

重复双击不会创建第二套服务。启动器会先确认当前端口上的服务确实是 DreamChord，健康运行时直接复用。

## 第一次体验建议

登录后先打开官方演示项目“第一根弦：消失的终页”。按下面顺序浏览：

1. 在首页查看项目卡片、章节数量和 Agent 入口。
2. 进入故事编辑器，观察左侧章节与场景、中间镜头卡、右侧实时预览。
3. 点击不同镜头卡，查看背景和人物如何沿剧情路径继承。
4. 打开剧情流程图，查看选项如何连接不同分支。
5. 点击完整预览，从故事入口开始试玩。
6. 进入创作 Agent，直接问“概括这个项目，还缺什么”。不选择章节也可以对话。
7. 进入全局素材库，查看立绘、背景和 CG 的组织方式。

## 创建第一段故事

1. 在首页新建项目，填写名称和简介。
2. 进入故事编辑器，创建章节和第一个场景。
3. 为场景选择背景并添加人物。
4. 新建对话或旁白镜头卡，填写文本。
5. 继续添加镜头时，背景和仍在场的人物会自动继承。
6. 新人物选择“登场 / 更新”；旧人物只有选择“退场”才会离开。
7. 需要玩家选择时添加选项，为每个选项指定后续场景。
8. 打开完整预览，确认文字、人物、背景和跳转。

编辑器会自动保存。页面显示保存冲突或失败时不要继续覆盖，先重新打开项目获取服务器上的最新版本。

## 导入自己的图片

素材属于账号的全局素材库，不属于某个章节。上传一次后，可以在不同故事中复用。

- 人物立绘优先使用透明背景 PNG。
- 白色、浅灰或纯色平底图可以使用本地去底处理。
- 房间、树林、街道等复杂背景不适合规则抠图，建议先使用专业分割工具。
- 原图始终保留，处理结果先作为候选图；检查边缘后再接受到素材库。
- 接受立绘后，在目标故事中绑定角色与表情，才能用于镜头卡。

详细要求见 [角色立绘规范](../SPRITE_ASSET_STANDARD.md) 和 [创作者工作流](CREATOR_WORKFLOW.md)。

## 使用创作 Agent

没有 API Key 时，本地 Agent 仍可回答时间、说明功能、概括项目、列出章节、盘点角色与素材、回顾对话和记忆、解释常见创作方法，并对“X 是什么”类问题进行中文 Wikipedia 只读查询。它还能生成项目上下文驱动的确定性续写草稿，并把用户或 Agent 的已选正文解析为可审核的场景、旁白、角色台词和选项。

开放式长文续写、对白润色和可应用剧情补丁建议在设置页配置 GLM、DeepSeek、Kimi、OpenAI 或兼容接口。普通讨论不必选择章节；只有需要把修改写入故事时才绑定章节。

Agent 的剧情或素材行动会先生成提案，不会直接修改。阅读差异或候选图后再选择应用、接受、拒绝或撤销。

## 常见问题

### 双击后窗口立即关闭

不要从压缩包预览窗口直接运行。先完整解压，再在项目根目录双击。如果仍然失败，打开 PowerShell，进入项目目录后运行：

```powershell
powershell -ExecutionPolicy Bypass -File .\start-dreamchord.ps1
```

这样可以看到完整错误信息。

### 出现 Prisma `EPERM unlink`

旧的 DreamChord 后端仍占用 Prisma 查询引擎。关闭之前启动的前端和后端命令窗口，再双击启动器。不要通过更换端口绕过文件锁。

### 页面没有自动打开

查看启动窗口末尾的“前端”地址，复制到浏览器。若启动器提示只有后端残留，关闭旧服务窗口后重新启动。

### 素材上传后找不到

到全局素材库查找。背景和 CG 可以从任意故事的镜头卡选择；立绘需要先接受候选，再在目标故事里绑定角色与表情。

### Agent 不能续写

先确认设置页已有可用模型和 API Key，再选择项目。只讨论内容不需要章节；要生成可应用的剧情修改时需要绑定章节。

## 自助诊断

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\doctor.ps1
```

重点查看带 `[FAIL]` 和 `[WARN]` 的中文提示。仍无法解决时，报告 Node.js 版本、完整错误文本、执行步骤，以及是否保留了已有数据库和上传目录；不要提交 `.env`、API Key 或数据库文件。
